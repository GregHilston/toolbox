# Pi Coding Agent Setup

Local-first AI coding agent ([pi-mono](https://github.com/badlogic/pi-mono)) backed by [oMLX](../omlx/).

## Quick Start

```bash
pi                              # start in current directory
pi --plan                       # start in plan mode (read-only exploration first)
pi @file.png "what's this?"     # vision input (Qwen3.6 supports it)
/plan                           # toggle plan mode mid-session
/todos                          # show plan progress
```

## How It's Deployed

| What | Where | How |
|------|-------|-----|
| `models.json`, `settings.json` | [pi.nix](../../nixos/modules/programs/tui/pi.nix) | Generated by home-manager |
| Local extensions | [dot/pi/.pi/agent/extensions/](../.pi/agent/extensions/) | Stowed to `~/.pi/agent/extensions/` |
| npm/git packages | [pi.nix](../../nixos/modules/programs/tui/pi.nix) `packages` option | `pi install` runs on nix activation |
| Sampling params (temp, top_k) | [model_settings.json](../omlx/.omlx/model_settings.json) | oMLX per-model overrides |
| Thinking budget (8192 tokens) | [model_settings.json](../omlx/.omlx/model_settings.json) | Prevents infinite reasoning loops |

## Tips

- **Thinking loops**: if the model spirals, the 8192-token thinking budget in oMLX will cut it off. You can also try `/nothink` or set `enable_thinking: false` per-request
- **Plan mode for complex tasks**: `/plan` forces the model to read the codebase before touching anything â€” big win for local models that tend to jump into edits prematurely
- **Keep system prompt lean**: pi's advantage is its tiny system prompt (<1000 tokens). Use `AGENTS.md` in project roots for per-project context, not global system prompt overrides
- **Update packages**: `pi update` pulls latest for all installed packages
